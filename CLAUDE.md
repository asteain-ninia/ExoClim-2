## プロジェクト概要

ExoClim-2 は、Worldbuilding Pasta の Worldbuilder's Log シリーズ（YouTube）と "An Apple Pie from Scratch" ブログ連載で解説される惑星気候設計手法を、インタラクティブなアプリとしてなぞって実装することを目的とする。

ITCZ → 風帯 → 海流 → 気流 → 気温 → 降水 → 気候帯（Köppen-Geiger）の順で導出する。

旧プロジェクト（`e:/WORKSPACE/ExoClim-/`）は仕様・実装ともに整合性を欠き、再構築が決定された（[実装済み.md](実装済み.md) 参照）。本リポジトリはゼロから再設計する。

## Claude の役割

- 主な担当: **設計支援・コードレビュー・コミット・ドキュメント更新**
- 広域の探索・調査は **Explore サブエージェント**に委譲し、主コンテキストを節約する
- 物理モデルの一次資料（Pasta ブログ）取得は WebFetch を活用する

## 作業開始時のチェック

- 実装/修正/仕様策定の着手前に [開発ガイド.md](開発ガイド.md) 5.1.2 の「AI作業開始ゲート」に従い、§6 の関連教訓を節番号・見出し・本文付きで引用し、今回の適用方針を明示する
- 不具合修正時は [実装済み.md](実装済み.md) の近い履歴と [開発ガイド.md](開発ガイド.md) §6 の対応表を確認する。根本原因が §6 にない場合は、履歴追記だけで済ませず §6 へ教訓化する
- Pasta準拠の論点では `docs/INTENT.md` の一次参照リンクから該当ブログを取得する
- 運用メモリ索引 `memory/MEMORY.md` を確認する（メモリと開発ガイドは役割が別物）

## コミット運用

- 粒度は機能単位で細かく
- コミットごとに 現状.md / 実装済み.md も更新する
- `git commit` は allowlist に入れない方針。メッセージ内容をユーザーが都度確認する

## 環境メモ（一時）

- Windows + Git Bash 環境。Read/Edit/Write のパス解決に失敗した場合は、**小文字ドライブ + 正スラッシュ**（例: `e:/WORKSPACE/ExoClim-2/...`）で試す。`cd` でプロジェクトルートに戻る操作は原則不要（cwd は常にプロジェクトルート）
