# Step 6: 降水

> **状態**: 骨格のみ。本文は Phase 2 で Pasta 一次資料から起こす。

## 1. 目的

惑星表面の月別降水量を、湿気の蒸発・移流・収束・地形性上昇から導出する。

## 2. 一次参照

> Phase 2 で WebFetch して該当節を抽出する。

- **Worldbuilder's Log #31 Precipitation** — <https://www.youtube.com/watch?v=lHV-jZUB5WU>（一次）
- **Worldbuilder's Log #32 Precipitation & Pressure Redo** — <https://www.youtube.com/watch?v=2RMyd9vo2Qk>（更新版）
- Worldbuilding Pasta — "An Apple Pie from Scratch, Part VIb" Step 6（Precipitation）

## 3. 入力

> Phase 2 で確定。暫定列挙:

- 風 U/V（Step 4 出力）
- 気温（Step 5 出力。飽和水蒸気量を決める）
- 海流（Step 3 出力。暖流上は蒸発増、寒流上は減）
- ITCZ 緯度線（Step 1 出力。直下バンド降水）
- 地形（標高・陸海分布）

## 4. アルゴリズム概要

> Phase 2 で Pasta 該当節を抜粋して再構成する。

論点:
- 海洋からの蒸発量（飽和水蒸気量 × 相対湿度 × 海流バイアス）
- 湿気の風方向 advection
- 内陸距離による減衰（Pasta の "2000km 内陸停止" など）
- 地形性上昇（orographic）による追加降水
- ITCZ 直下の収束帯降水
- 標高による降水カット

## 5. 出力

> Phase 2 で確定。暫定:

- 月別の降水量
- 月別の残存水蒸気量
