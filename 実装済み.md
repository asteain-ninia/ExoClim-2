### 設計判断（P4-1b）

- **単位系の識別子表記**: `lapseRateCelsiusPerKm` / `surfaceGravityMps2` / `oceanMixedLayerDepthMeters` 等、[開発ガイド.md §2.2.3](開発ガイド.md) に従い物理量の識別子に単位を含める。角度は度（Deg）を既定。
- **温室効果と熱輸送は無次元相対値（地球 = 1.0）**: Pasta が単一スライダーで露出している粒度に合わせる（[要件定義書.md §2.1.3](要件定義書.md)「Pasta が単一スライダーで扱う量を本ツールが過細分化することは避ける」）。`greenhouseStrengthRelative` / `meridionalHeatTransportRelative` / `zonalHeatTransportRelative` の 3 量。

- **気温減率は Pasta 既定 4.46 °C/km**: NASA の標準値 6.5 °C/km ではなく [docs/spec/05_気温.md §4.3](docs/spec/05_気温.md) の Pasta 値を採用。地球プリセットは「Pasta が地球で使う値」の再現を目的とするため。
- **`TerrainSource` は識別情報のみ保持**: 実体データ（標高・陸海・大陸 ID マップ）は本識別子を入力に地形前処理層（P4-1c 以降）が解決する。これにより設定スナップショット（[要件定義書.md §4.4](要件定義書.md) `Snapshot`）を JSON で軽量に保てる。`procedural` バリアントの制御パラメータは現時点で `seed` と `landFraction` の 2 項目のみ（Phase 4 の地形生成器実装時に必要なら拡張）。
- **`PlanetParams` は 4 区画の構成体**: 派生量・中間結果は本型に含めない（[技術方針.md §2.2.1](技術方針.md) 入出力契約）。
- **全フィールド `readonly`**: ドメイン層の不変性を型レベルで担保（grid.ts と同じ方針）。
- **数値根拠の明示**: NASA Earth Fact Sheet（J2000.0）と Pasta の引用箇所をソースコメントに記載。利用者が後から検証できる根拠を残す。

## 2026-04-28 Phase 4-1a: ドメイン層 `Grid` / `Cell` 型と既定 1° 解像度生成器

P4-1a を完了。[要件定義書.md §4.1](要件定義書.md) の「グリッド・セル」識別子を最初のドメイン型として確立した。

### 追加・更新ファイル（P4-1a）

- 新規: [tests/unit/domain/grid.test.ts](tests/unit/domain/grid.test.ts) — 12 件の単体テスト

### 設計判断（P4-1a）

- **解像度の型レベル固定**: `GridResolutionDeg = 0.5 | 1 | 2` を厳密 union とし、要件定義書 §4.1 の「三段階切替」を型として写像。任意の数値解像度を許さない。
- **既定 1°**: `DEFAULT_GRID_RESOLUTION_DEG = 1` を定数として export。`createGrid()` は引数省略で 180×360 = 64,800 セルを返す。
- **`Cell` 属性は §4.1 の固定 5 項目**: `latitudeDeg` / `longitudeDeg` / `elevationMeters` / `isLand` / `continentId`。識別子に単位を含める ([開発ガイド.md §2.2.3](開発ガイド.md))。海洋セルでは `continentId: null`（`exactOptionalPropertyTypes` 下で optional ではなく `string | null`）。
- **地形なしのデフォルト初期化**: `createGrid` は座標格子のみを担い、全セルを「海・標高 0・大陸所属 null」で初期化する。地形ソース (`TerrainSource`) の適用は P4-1b 以降の責務として明確に分離。
- **セル中心の決定的計算**: 緯度 `-90 + (i+0.5)·dRes`、経度 `-180 + (j+0.5)·dRes`。同入力 → 同出力（[要件定義書.md §3.2](要件定義書.md)）。
- **全フィールド `readonly`**: ドメイン層の不変性を型レベルで担保。
